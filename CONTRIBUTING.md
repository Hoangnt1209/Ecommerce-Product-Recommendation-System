# Team Roles & Project Responsibilities

Course: **DDM501 - AI in Production: From Models to Systems**  
Project: **Real-Time E-Commerce Product Recommendation System**

## Team Structure & Task Allocation

| Team Member | Role | Key Responsibilities |
|---|---|---|
| **Member 1** | Lead ML & System Architect | Data preprocessing pipeline, SVD Matrix Factorization, Hybrid Ensemble design, System Architecture |
| **Member 2** | Deep Learning & MLflow Engineer | PyTorch Neural Collaborative Filtering (NCF) model, MLflow experiment tracking, artifact & weights registry |
| **Member 3** | Backend & MLOps Engineer | FastAPI REST API development, Prometheus & Grafana metrics, Docker & Docker Compose containerization |
| **Member 4** | Responsible AI & QA Specialist | Model explainability (feature relevance), Fairness & bias audit, PyTest test suite, CI/CD GitHub Actions |

## Workflow & Git Guidelines
1. Feature branches: `feature/ml-models`, `feature/fastapi-serving`, `feature/mlflow-tracking`.
2. Commit convention: `feat: ...`, `fix: ...`, `docs: ...`.
3. Pull requests require approval and passing green CI/CD pipeline before merging to `main`.
